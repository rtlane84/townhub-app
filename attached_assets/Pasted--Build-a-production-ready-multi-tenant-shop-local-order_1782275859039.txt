
Build a production-ready multi-tenant “shop local” ordering marketplace SaaS.

App concept:
A small-town local ordering hub where customers can browse participating local businesses, click a business, view that business’s storefront, order items for pickup or local delivery, and pay online. Think “local marketplace homepage + individual business storefronts.”

Working name:
LocalOrderHub

Primary example businesses:
1. Flower shop
2. Diner/cafe
3. Greenhouse / small market

Core idea:
Version A marketplace:
- Main public homepage lists all participating businesses
- Each business has its own storefront page inside the platform
- Customers can order from one business at a time
- Each business has its own admin dashboard
- Platform owner has a master admin dashboard

Tech requirements:
- Use Next.js 14 App Router
- TypeScript
- Tailwind CSS
- Prisma ORM
- PostgreSQL
- NextAuth/Auth.js for authentication
- Stripe for payments, but make it easy to run in test mode
- Zod for validation
- Responsive mobile-first design
- Clean README with setup instructions
- Seed data for demo businesses/products/orders
- Modern rural/small-town design, clean and professional

User roles:
1. Customer
   - Can browse businesses
   - Can view storefronts
   - Can add items to cart
   - Can checkout
   - Can receive order confirmation

2. Business Owner
   - Can log in
   - Can manage only their own business
   - Can manage products/categories
   - Can view orders
   - Can update order status
   - Can edit business profile/settings

3. Platform Admin
   - Can manage all businesses
   - Can create/edit/disable businesses
   - Can assign business owners
   - Can view all orders
   - Can view basic platform stats

Public pages:
1. Homepage
   - Hero section: “Order Local. Support Local.”
   - Short subtitle about ordering from local businesses
   - Featured businesses
   - Categories: Food, Flowers, Plants & Market
   - Today’s specials section
   - Call-to-action for businesses: “List your business”

2. Business directory page
   - List all active businesses
   - Filter by business type
   - Search by business name
   - Cards should show:
     - logo
     - business name
     - type
     - pickup/delivery availability
     - open/closed status
     - short description

3. Business storefront page
   URL pattern: /business/[slug]
   - Business logo/name
   - description
   - address/phone/hours
   - pickup/delivery badges
   - product categories
   - product cards
   - add-to-cart buttons
   - business-specific ordering fields at checkout based on business type

4. Cart/checkout page
   - Customer can only checkout from one business at a time
   - Show cart items
   - Customer info:
     - name
     - email
     - phone
   - Fulfillment:
     - pickup or delivery
     - pickup time/date when applicable
     - delivery address when applicable
   - Notes field
   - Business-type fields:
     Florist:
       - recipient name
       - card message
       - occasion type
       - delivery date
     Diner:
       - pickup time
       - special instructions
     Greenhouse:
       - pickup date
       - substitute if sold out checkbox
   - Stripe checkout integration
   - Also allow “pay at pickup” if business has that enabled

5. Order confirmation page
   - order number
   - business name
   - order summary
   - pickup/delivery details
   - customer confirmation message

Business owner dashboard:
Route prefix: /dashboard/business

Pages:
1. Overview
   - today’s orders
   - upcoming orders
   - recent revenue
   - pending orders

2. Orders
   - list orders for that business only
   - filter by status/date
   - order statuses:
     - NEW
     - CONFIRMED
     - PREPARING
     - READY_FOR_PICKUP
     - OUT_FOR_DELIVERY
     - COMPLETED
     - CANCELED
   - order detail page
   - update status button

3. Products
   - create/edit/delete products
   - mark available/sold out
   - upload/store image URL
   - product fields:
     - name
     - description
     - price
     - category
     - imageUrl
     - available
     - featured
     - prepTimeMinutes

4. Categories
   - create/edit/delete categories

5. Business Settings
   - business name
   - slug
   - logo URL
   - hero image URL
   - description
   - address
   - phone
   - hours
- Business types:
        - FOOD_VENDOR: diners, food trucks, bakeries, catering
	- FLORIST: flower shops, funeral flowers, prom flowers
	- GARDEN_MARKET: greenhouses, produce stands, small grocery/marker
	- RETAIL_STORE: hardware stores, gift shops, general retail, cannabis store menu/reservation mode
	- BUILDING_SUPPLY: lumber yards, contractor materials, quote requests
	- SERVICE_PROVIDER: salons, barbers, IT services, repair shops
	- FUNERAL_SERVICE: funeral homes, memorial service-related requests

   - pickup enabled

   - minimum order

- Business
- BusinessOwner mapping if needed

- Order
- OrderItem
- Customer

Important database requirements:
- Every Product belongs to a Business
- Every Category belongs to a Business
- Every Order belongs to one Business
- Business owners can only access their own business data
- Admin can access everything
- Use enum for Role
- Use enum for BusinessType
- Use enum for OrderStatus
- Use enum for FulfillmentType
